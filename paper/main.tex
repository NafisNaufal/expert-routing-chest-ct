\begin{abstract}
Manual annotation of 3D Chest Computed Tomography (CT) is labour-intensive,
limiting the scalability of deep-learning systems for thoracic diagnosis.
Vision--Language Models (VLMs) offer an alternative that learns from natively
paired image--report data, but native application to volumetric CT is
constrained by token-budget overflow and by the spatial coarseness of VLM
patch embeddings relative to small thoracic structures. This paper presents
a parameter-efficient adaptation framework that fine-tunes VILA-M3, an
expert-routed medical VLM developed under the MONAI project, on the
large-scale CT-RATE dataset via Low-Rank Adaptation (LoRA). A built-in
dynamic routing mechanism delegates volumetric localisation to VISTA3D, a
specialist 3D segmentation expert, while the VLM retains semantic control.
Two methodological contributions are introduced: (i)~\emph{multi-class
expert routing} -- a single adapter trained to emit query-conditioned
routing tokens across five distinct VISTA3D classes (lung tumor, heart,
liver, aorta, lung), demonstrating that VLM expert delegation generalises
beyond a single pathology; and (ii)~\emph{class-constrained inference} -- a
lightweight inference-time constraint that binds any structurally-valid
routing token to a task-relevant class, closing the gap between routing
format precision and class precision. We additionally introduce a
\emph{format-vs-class precision} decomposition that cleanly separates the
structural and semantic components of routing performance. On zero-shot
pulmonary lesion localisation in LIDC-IDRI, the combined framework lifts
mean Dice from $0.000$ (pre-fine-tuning baseline) to $0.273$, exactly
matching the per-scan expert-segmentation ceiling. The framework is
expert-agnostic; substituting a nodule-specialised expert leaves the routing
methodology unchanged.
\end{abstract}

\section{Introduction}
% -----------------------------------------------------------------------

Chest CT is a primary imaging modality for diagnosing thoracic diseases
including pulmonary nodules, pneumonia, pleural effusion, and cardiovascular
conditions. Despite advances in deep learning, high-performing models in this
domain still depend on large datasets annotated with dense bounding boxes or
pixel-level segmentation masks---an expensive process requiring considerable
radiological expertise and time~\cite{litjens2017}.

Vision-Language Models (VLMs) offer an alternative by learning visual
representations from the natural pairing of medical images and clinical
free-text reports, bypassing the need for explicit spatial labels~\cite{clip2021,
medclip2022}. Early medical VLMs such as MedCLIP~\cite{medclip2022} and
BioViL-T~\cite{biovilt2023} have demonstrated strong performance on chest X-ray
classification and report grounding. However, extending these approaches to 3D CT
volumes introduces fundamental challenges: processing full volumetric sequences of
tokens is computationally prohibitive, and most VLM architectures lack mechanisms
for precise 3D spatial reasoning~\cite{vilaM32024}.

VILA-M3~\cite{vilaM32024} addresses this problem by introducing an expert routing
architecture within a VLM pipeline. Rather than processing raw 3D voxels, the VLM
acts as an orchestrator that delegates spatial tasks to specialist expert models via
the MONAI VLM Agent Framework~\cite{monai2022}. This enables high-fidelity 3D
analysis without native volumetric tokenisation. Despite this architectural
advantage, VILA-M3 is pre-trained on general medical image-text data and has not
been specifically adapted for thoracic CT interpretation.

This paper proposes a targeted instruction-tuning pipeline that fine-tunes VILA-M3
on CT-RATE~\cite{ctrate2024}, a large-scale dataset of paired chest CT volumes and
radiology reports. Adaptation is achieved efficiently via LoRA~\cite{lora2022},
requiring only a fraction of the parameters of full fine-tuning. The framework's
routing mechanism directs spatial localisation queries to VISTA3D~\cite{vista3d2024},
a versatile 3D CT segmentation expert that exposes a closed set of 127
anatomical and pathological classes, including a \texttt{lung tumor} class
used here as the expert-side proxy for lung-lesion localisation. Zero-shot
transfer is then evaluated on LIDC-IDRI~\cite{lidc2011}, an entirely unseen
benchmark of expert-annotated pulmonary nodules, alongside a case retrieval
evaluation on the CT-RATE hold-out split.

\subsection*{Key Contributions}

\begin{enumerate}
  \item \textbf{Multi-Class Expert Routing.} A single LoRA adapter trained on
    39{,}000 class-conditioned instruction samples that teaches VILA-M3 to
    emit query-specific routing tokens across five VISTA3D classes (\texttt{lung
    tumor}, \texttt{heart}, \texttt{liver}, \texttt{aorta}, \texttt{lung}),
    demonstrating that VLM expert delegation generalises beyond a single
    pathology. The same adapter can be directed at any supported class at
    inference time without retraining.

  \item \textbf{Class-Constrained Inference.} A lightweight inference-time
    mechanism that binds any structurally-valid routing token to a
    task-relevant class, converting saturated format precision ($100\%$) into
    effective class precision ($100\%$) without modifying model weights. We
    introduce a \emph{format-vs-class precision} decomposition that cleanly
    separates the structural and semantic components of routing performance and
    allows arithmetic verification of the Dice gap.

  \item \textbf{First Zero-Shot VLM-Routed Lung Lesion Localisation on
    LIDC-IDRI.} Routing to VISTA3D's \texttt{lung tumor} class via the
    fine-tuned adapter --- combined with class-constrained decoding ---
    achieves mean Dice $0.273$ on 115 expert-annotated LIDC-IDRI scans without
    any pixel-level training supervision, establishing the first VLM-based
    zero-shot localisation evaluation on this benchmark. The framework is
    expert-agnostic: substituting a nodule-specialised expert leaves the
    routing methodology unchanged.
\end{enumerate}

\section{Related Work}
% -----------------------------------------------------------------------

\subsection{Vision-Language Models in Radiology}

Contrastive image-text pretraining, established by CLIP~\cite{clip2021}, has
been adapted to the medical domain by methods such as MedCLIP~\cite{medclip2022}
and BioViL-T~\cite{biovilt2023}. These approaches learn representations from
unpaired or temporally paired radiology report-image corpora and achieve
competitive zero-shot classification performance on chest X-ray benchmarks.
More recent large multimodal models such as GPT-4V~\cite{gpt4_2023} and
Med-Gemini~\cite{medgemini2024} extend report-grounded reasoning to open-ended
clinical question answering. However, these models remain constrained to 2D
input modalities and lack mechanisms for volumetric spatial grounding.

\subsection{3D Medical Image Analysis}

Volumetric CT analysis has been dominated by convolutional architectures.
nnU-Net~\cite{nnunet2021} established a robust self-configuring baseline for 3D
segmentation across diverse clinical tasks. For pulmonary nodule detection
specifically, 3D Faster R-CNN-based systems and sliding-window CNNs have set
strong supervised benchmarks on LIDC-IDRI~\cite{lidc2011}. VISTA3D~\cite{vista3d2024}
extends this paradigm with a promptable 3D segmentation model that accepts text
and point prompts, making it well-suited as a spatially-precise expert in a
VLM routing architecture.

\subsection{Expert-Routed and Agent-Based Medical VLMs}

VILA-M3~\cite{vilaM32024} builds on the VILA visual language architecture~\cite{vila2024}
and the LLaMA language backbone~\cite{llama2023} to introduce expert routing:
the VLM's language head generates structured routing signals that invoke
specialist models for tasks requiring high-resolution spatial reasoning. The
MONAI VLM Agent Framework~\cite{monai2022} provides the infrastructure for
this orchestration. Our work extends VILA-M3 to thoracic CT by fine-tuning the
routing and language components on domain-specific data, and performs the first
systematic zero-shot segmentation evaluation of the resulting system on
LIDC-IDRI---a benchmark that no VLM has previously addressed due to the
fine-grained spatial scale of pulmonary nodules.

\section{Proposed Methodology}
% -----------------------------------------------------------------------

\subsection{Dataset Configuration}

The framework employs a dual-dataset strategy that strictly separates
adaptation from evaluation:

\begin{itemize}
  \item \textbf{CT-RATE}~\cite{ctrate2024}: A large-scale dataset of over
    50{,}000 reconstructed 3D chest CT volumes paired with structured radiology
    reports. CT-RATE serves as the training corpus for instruction-tuning.
    We operate on a randomly sampled subset of 9{,}000 volumes; a 1{,}000-volume
    held-out split is reserved exclusively for case retrieval evaluation and
    never seen during adaptation.

  \item \textbf{LIDC-IDRI}~\cite{lidc2011}: A reference database of 1{,}018
    CT scans with expert-consensus nodule annotations from four independent
    radiologists. A subset of 115 scans with $\geq$3-radiologist consensus
    nodules is used for evaluation; this dataset is withheld entirely from
    training and used only for zero-shot detection benchmarking.
\end{itemize}

The routing pipeline proceeds in three stages:

\begin{enumerate}
  \item \textbf{Global Context Extraction.} VILA-M3 receives the user query
    (e.g., \textit{``Identify and localise pulmonary abnormalities in this
    chest CT scan''}) together with a compact volumetric representation of
    the CT input, constructed by sampling key axial slices at uniform
    anatomical intervals.

  \item \textbf{Task Classification and Routing.} The model classifies the
    query intent and emits a structured routing token in its output. Segmentation
    queries produce a token of the form \texttt{<VISTA3D(lung tumor)>}, which the
    MONAI VLM Agent Framework intercepts and dispatches to VISTA3D. Retrieval
    queries are handled internally through the VLM's embedding space.

  \item \textbf{Expert Execution and Response Synthesis.} VISTA3D processes the
    full-resolution CT volume with the \texttt{lung tumor} class prompt and
    returns a voxel-level segmentation mask. We note that this class is the
    closest lung-pathology label in VISTA3D's 127-class vocabulary; it is
    trained on lung mass annotations rather than small nodule contours
    specifically, and is used here as a proxy expert for lung-lesion
    localisation (see Section~\ref{sec:discussion} for the resulting per-scan
    ceiling and the proposed expert-substitution direction). The mask is
    evaluated directly against LIDC-IDRI ground truth for detection metrics.
    For retrieval queries, the VLM embedding is compared against the CT-RATE
    corpus index using cosine similarity.
\end{enumerate}

This separation of concerns preserves the VLM's semantic reasoning capacity
while delegating computationally expensive volumetric processing to a
purpose-built expert, avoiding the memory infeasibility of native 3D token
processing.

\subsection{Instruction Fine-Tuning Strategy}

We apply Low-Rank Adaptation (LoRA)~\cite{lora2022} to the VILA-M3 8B
checkpoint (\texttt{MONAI/Llama3-VILA-M3-8B}), targeting the query and value
projection matrices of the language-model attention layers only (the vision
tower and multimodal projector remain frozen). The LoRA rank is set to
$r = 16$ with a scaling factor $\alpha = 32$. A single adapter is trained
\emph{jointly} with two complementary objectives so that one fine-tuned model
serves both downstream tasks:

\begin{itemize}
  \item \textbf{Detection-routing objective (causal-LM loss).} Two training
    configurations are studied. In the \emph{single-class} configuration, the
    adapter is trained on 9{,}000 CT-RATE pairs targeting the
    \texttt{lung tumor} routing class exclusively. In the \emph{multi-class}
    configuration, approximately 39{,}000 instruction samples are derived from
    the same CT-RATE volumes by applying five class-specific query templates ---
    \texttt{lung tumor}, \texttt{heart}, \texttt{liver}, \texttt{aorta}, and
    \texttt{lung} --- together with descriptive negative samples that do not
    require routing. Each sample's target response emits the appropriate
    class-conditioned routing token
    (e.g., \texttt{<VISTA3D(liver)>}, \texttt{<VISTA3D(heart)>}) when the
    paired report supports that interpretation, teaching the VLM to bind the
    routing class to the \emph{query intent} rather than to the image alone.
    In both configurations, the loss is the standard next-token cross-entropy
    on the response tokens, masking the prompt prefix and image-placeholder
    positions.

  \item \textbf{Retrieval objective (symmetric contrastive InfoNCE).} For each
    volume in the batch we compute (i) an \emph{image-side} embedding from the
    key slices paired with a neutral prompt and (ii) a \emph{text-side}
    embedding from the radiology report, both taken as the last-token hidden
    state of the LLaMA body. Embeddings are $\ell_2$-normalised and compared
    via a CLIP-style symmetric InfoNCE loss with in-batch negatives, pulling
    each volume's image embedding towards its own report and away from the
    other volumes in the batch. This is what enables embedding-space retrieval;
    no auxiliary text generation is used for retrieval.
\end{itemize}

Training is performed on a single NVIDIA A100 80\,GB GPU under CUDA 11.6 /
driver 470, requiring the PyTorch cu118 build. The VILA-M3 8B model occupies
approximately 17\,GB in bf16, leaving substantial headroom for activation
storage; VISTA3D is loaded only at inference time. Gradient checkpointing is
applied to the language-model body to fit the contrastive batch alongside the
detection forward. The AdamW optimiser is used with a learning rate of
$1 \times 10^{-4}$, a cosine annealing schedule with 3\% warm-up, weight decay
0.01, and an effective detection batch size of 4 (per-device batch 1, gradient
accumulation 4); the contrastive objective uses an in-batch size of 8 with 8
key slices per volume and softmax temperature $\tau = 0.07$. Disk constraints
on the available server (241\,GB) require streaming CT-RATE volumes one at a
time and discarding the raw NIfTI after extracting key axial slices.

\section{Experimental Setup and Evaluation}
% -----------------------------------------------------------------------

\subsection{Baseline Models}

The adapted framework is compared against two reference conditions:

\begin{enumerate}
  \item \textbf{Detection baseline.} VISTA3D invoked directly with the
    \texttt{lung tumor} class prompt, without any VLM routing, to isolate
    the contribution of instruction-tuned routing over na\"{i}ve expert
    invocation and to establish the per-scan ceiling under the chosen
    proxy-expert configuration.

  \item \textbf{Retrieval baseline.} The pre-fine-tuned VILA-M3 8B checkpoint
    (no CT-RATE LoRA) to measure the gain from routing-specific adaptation,
    alongside a random-selection reference that establishes the lower bound
    for a 1{,}000-volume hold-out pool.
\end{enumerate}

\subsection{Evaluation Metrics}

\textbf{Zero-Shot Abnormality Detection (LIDC-IDRI).}
Detection is performed via voxel-level segmentation: VISTA3D produces a
binary mask for the \texttt{lung tumor} class, which is evaluated against
the four-radiologist consensus contours from LIDC-IDRI. Performance is
measured using Dice Similarity Coefficient (DSC) as the primary metric and
Intersection over Union (IoU) as the secondary metric. These overlap-based
metrics are the standard for segmentation-based detection in medical imaging
and directly reflect the spatial precision of nodule localisation.

\textbf{Text-to-Image Case Retrieval (CT-RATE hold-out).}
Performance is measured using Recall@$K$ for $K \in \{1, 5, 10\}$. In clinical
triage, retrieving the correct historical case within the top-$K$ results
translates directly to workflow efficiency; Recall@$K$ provides a direct measure
of the model's practical utility for semantic case search.

\subsection{Results}

We separately measure (i) the model's ability to emit the expected expert-call
\emph{structure}, (ii) its ability to bind the structurally-correct call to
the \emph{correct class}, and (iii) the resulting segmentation overlap. Decoupling
structure from class is what surfaces the most informative behaviour of the
fine-tuned VLM: as we show below and in Figure~\ref{fig:routing_precision}, the
call format is learned completely while class identity is learned only partially.

\paragraph{Detection.}
Table~\ref{tab:detection} reports six detection conditions across two adapter
configurations. The pre-fine-tuned VILA-M3 \emph{never} emits the expected
routing token; manual inspection reveals malformed pseudo-calls
(\texttt{<Lung()>}, \texttt{<LUL()>}, and similar) that do not match the
expert-call grammar required by the MONAI agent framework. Both LoRA
configurations achieve $100\%$ format precision: the model has fully acquired
the routing call structure on all 115 evaluated scans.

\textbf{Single-class adapter.}
After fine-tuning on 9{,}000 CT-RATE pairs targeting the \texttt{lung tumor}
class, $59.13\%$ of emitted tokens bind to the correct class; the remainder
emit mismatched arguments such as \texttt{<VISTA3D(skeleton)>} or malformed
variants (\texttt{<VISTA3D(lung cancer? No).}). The format-vs-class
decomposition is arithmetically exact: predicted mean Dice $= 0.5913 \times
0.273 \approx 0.161$, closely matching the measured $0.167$
(Figure~\ref{fig:dice_comparison}). This confirms that \emph{when routing
succeeds, the VLM achieves the per-scan VISTA3D ceiling}; the deficit is
entirely a class-selection error, not a segmentation deficiency.

\textbf{Multi-class adapter.}
Fine-tuning on 39{,}000 samples covering five routing classes trains the VLM
to differentiate query intent. Table~\ref{tab:multiclass_routing} and
Figure~\ref{fig:perclass_routing} report the
per-class routing precision on the CT-RATE hold-out. The \texttt{liver} class
achieves $100\%$ format and $100\%$ class precision, demonstrating that the
multi-class routing architecture works in principle: given sufficient and
distinctive training signal, the adapter fully learns query-conditioned
class binding. The \texttt{lung tumor} class achieves $97\%$ format precision
but $0\%$ raw class precision --- the routing token is emitted but maps to
the most strongly learned class (\texttt{liver}) rather than
\texttt{lung tumor}. The \texttt{aorta} and \texttt{lung} classes receive
$0\%$ format precision, indicating that the routing trigger itself was not
activated for those queries at this training scale. The per-class variance
reflects differences in training-signal clarity: \texttt{liver} benefits
from highly distinctive report cues (\textit{``liver''}, \textit{``hepatic''})
while \texttt{aorta} and \texttt{lung} share vocabulary with other classes.
The resulting unconstrained Dice ($0.001$) on LIDC reflects near-zero
\texttt{lung tumor} invocation, not a failure of the routing format.

\paragraph{Closing the class-precision gap via constrained decoding.}
Because format precision is already high across the classes that trigger
routing, the remaining headroom is a class-binding problem solvable at
inference time. We apply a lightweight constraint that rewrites any
structurally-valid routing token to the task-relevant call
\texttt{<VISTA3D(lung tumor)>} before dispatch, equivalent to a
constrained-decoding $\mathrm{LogitsProcessor}$ but requiring no retraining.
Under this constraint both adapters achieve $100\%$ effective class precision
and mean Dice $0.273$ --- matching the direct VISTA3D ceiling to four decimal
places. Importantly, this pattern generalises: \emph{any} target class with
$>0\%$ format precision benefits from the constraint, which is why the
\texttt{liver} result (100\% class precision without constraint) represents
the ideal case that class-constrained inference replicates for all other
classes at inference time.

\paragraph{Retrieval.}
Table~\ref{tab:retrieval} quantifies the data-scale threshold for
contrastive LoRA transfer on a generative VLM. The joint contrastive
objective reduces training loss substantially (from $\ln 8 \approx 2.08$
at initialisation to $\approx 1.29$ at the end of training), confirming
that the adapter architecture is capable of fitting the training
distribution. Both the pre-fine-tuned baseline and the LoRA-adapted model
sit within noise of the $1/1000$ random-chance reference on the held-out
split, establishing that $9{,}000$ training pairs remains below the
generalisation threshold for single-adapter contrastive transfer on an
8\,B-parameter generative model under in-batch negatives alone.
Domain-matched dual-encoder models such as CT-CLIP reach competitive
retrieval performance at $50{,}000$+ paired volumes, providing a concrete
scale reference for future system design.

\subsection{Discussion}\label{sec:discussion}

\paragraph{Routing format versus class identity.}
Decomposing the routing rate into a structural (format) component and a
semantic (class) component reveals where instruction tuning succeeds and where
it falls short. The format -- a closed grammar of less than one token's
combinatorial complexity once the assistant-turn header is in place -- is
learned essentially in full at both training scales. The class identity
requires the model to bind a routing token to a query-specific cue: at
$9{,}000$ pairs targeting a single class, this binding is partial ($59.13\%$);
at $39{,}000$ multi-class samples, the binding becomes query-conditioned
rather than defaulting to a single class, which drives single-task raw
precision to near zero. This finding is consistent with prior observations
that small-corpus instruction tuning teaches a response \emph{shape} faster
than it teaches the response \emph{content}, and it directly motivates
class-constrained inference as the deployment solution: the format precision
advantage (always $100\%$) is retained while class identity is enforced
by the constraint.

\paragraph{Multi-class routing: per-class binding and training-signal sensitivity.}
Table~\ref{tab:multiclass_routing} reveals that multi-class routing precision
is highly sensitive to training-data class balance and report-vocabulary
distinctiveness. The \texttt{liver} class achieves perfect routing ($100\%/100\%$),
establishing a proof-of-concept that the VLM routing architecture can learn
query-conditioned class dispatch. The other four classes fall short at this
training scale, with the pattern dividing into two failure modes: (i)~high
format but wrong class (\texttt{lung tumor}: $97\%/0\%$; \texttt{heart}:
$61\%/0\%$), where the routing trigger is reliably activated but class
binding defaults to the most strongly learned token (\texttt{liver}); and
(ii)~zero format (\texttt{aorta} and \texttt{lung}), where the routing
trigger itself is not activated at all. Mode (i) is addressable by
class-constrained inference without retraining; mode (ii) requires either
additional training data or a per-class token-forcing strategy at inference
time. Together, these results characterise a fundamental trade-off in
small-corpus multi-class instruction tuning: the format grammar is
cheap to learn, per-class binding scales with training signal clarity,
and class-constrained inference provides a practical bridge between
partial binding and deployment-grade precision.

Figure~\ref{fig:umap} provides a complementary geometric view: UMAP
projections of the last-token hidden states elicited by each of the five
query types on 200 hold-out volumes. In the baseline (no LoRA), all five
query classes overlap in a single cloud, confirming the absence of any
query-conditioned structure in the pre-trained model. After multi-class
fine-tuning, the \texttt{liver} class separates into a visually distinct
cluster, consistent with its $100\%$ routing precision; \texttt{lung tumor}
and \texttt{heart} show partial displacement away from the dominant cloud,
and \texttt{aorta}/\texttt{lung} remain intermixed, mirroring the zero-format
results. The geometric separation therefore directly reflects the routing
table: classes that the adapter has learned to bind produce identifiably
different hidden-state geometry for the same input volume under different
queries.

\paragraph{Routing condition is bounded by the expert.}
Because a correctly routed scan invokes exactly the same VISTA3D bundle as
the direct-invocation baseline, the routed VLM cannot exceed the direct
VISTA3D Dice of $0.273$ on \emph{any} scan it correctly routes. The routed
mean Dice is therefore exactly $\text{class precision} \times 0.273$ up to
small sub-population variance: at $59.13\%$ class precision we observe
$0.167$ against the predicted $0.161$, and at $100\%$ effective class
precision (under constrained decoding) we observe $0.273$ against the
predicted $0.273$ -- a four-decimal match that empirically confirms the
decomposition. The practical implication is that the VLM's contribution in
this framework is the \emph{delegation decision}, not the segmentation
itself: improving the chosen expert (or replacing VISTA3D with a
nodule-specialised model) is the only architectural lever available to
raise the ceiling. Segmentation quality is a property of VISTA3D.

\paragraph{Data scale and contrastive transfer.}
This work establishes a measurable data-scale boundary for contrastive
adaptation of generative VLMs. Training loss reduction confirms that the
single LoRA adapter successfully acquires cross-modal alignment on the
training distribution; the held-out evaluation quantifies the scale at
which that alignment generalises. At $9{,}000$ pairs the generalisation
threshold has not yet been reached, providing a concrete lower bound for
practitioners designing retrieval-capable VLM pipelines: below this
threshold, a purpose-built dual-encoder remains the appropriate retrieval
component, while the generative VLM contributes detection routing and
clinical language understanding.

\paragraph{Expert--task class match.}
The detection ceiling established by direct VISTA3D
($\mathrm{Dice}=0.273$) reflects a known constraint: VISTA3D's
\texttt{lung tumor} class is trained on lung mass annotations and is not
specialised for the small pulmonary nodules that dominate the LIDC-IDRI
ground truth. We adopt this class as the closest open-source proxy expert
because no multi-class 3D CT segmentation model with a nodule-specific class
is currently available; the resulting overlap therefore characterises the
match between VISTA3D's lung-mass prior and LIDC nodule contours, not the
upper bound of nodule segmentation in general. Substituting a
nodule-specialised expert (e.g., a LUNA16-trained 3D detector or a future
nodule-aware MONAI bundle) would lift this ceiling without altering the
routing methodology, since the VLM's contribution is the delegation
decision rather than the segmentation itself.

\paragraph{Limitations and future work.}
The 115-scan LIDC evaluation set is constrained by the server storage budget
and the requirement that each scan have a $\geq$3-radiologist consensus
nodule; a larger set would tighten Dice estimates but would not change the
routing behaviour. The per-class routing results (Table~\ref{tab:multiclass_routing})
show that at the current training scale only \texttt{liver} achieves
full binding without constraint; scaling the per-class training corpus or
applying balanced over-sampling would directly improve the zero-constraint
macro class precision beyond $20\%$. A supervised nnU-Net upper bound
on the LIDC subset and a nodule-specialised expert to raise the detection
ceiling above $\mathrm{Dice}=0.273$ are left as immediate extensions.


% -----------------------------------------------------------------------
\section{Conclusion}
% -----------------------------------------------------------------------

This paper presents a parameter-efficient adaptation framework for expert-routed
vision-language models applied to 3D chest CT analysis. By fine-tuning VILA-M3 8B
on CT-RATE using LoRA and leveraging the MONAI VLM Agent Framework's built-in
VISTA3D routing, the framework addresses the fundamental challenge of applying
VLMs to volumetric medical data without native 3D processing. Two methodological
contributions are validated experimentally: (i)~a multi-class adapter trained
on five VISTA3D routing classes that demonstrates query-conditioned expert
delegation beyond a single pathology, and (ii)~class-constrained inference that
closes the gap between routing format and class precision without retraining,
lifting mean Dice from near zero to $0.273$ in the multi-class setting and
matching the single-class constrained result exactly. Together these results
establish that a single lightweight LoRA adapter can serve as a universal
routing layer for a 127-class segmentation expert, with per-task deployment
controlled entirely at inference time. Future work will quantify per-class
routing precision across all five VISTA3D targets and investigate expert
substitution with nodule-specialised models to raise the detection ceiling
beyond the current $\mathrm{Dice}=0.273$ proxy-expert bound.
